# KPI TIME TRACKING SYSTEM

Allow users to generate charts for **any KPI available in the system**.

Users must be able to select a KPI from the list and visualize its evolution over time.

Add a **time tracking selector** allowing the user to display KPI trends by:

* Day (Jour)
* Week
* Month

The system must automatically aggregate data according to the selected time scale.

Example:

Daily view → show daily values
Weekly view → show weekly averages or totals
Monthly view → show monthly aggregation

---

# KPI CHART GENERATION

When a KPI is selected, automatically generate a chart showing its evolution over time.

Supported chart types:

* Line chart (default for time tracking)
* Bar chart
* Area chart
* Stacked bar chart
* Pie chart (for distribution analysis)

Users must be able to switch chart types dynamically without reloading data.

---

# KPI TRACKING FEATURES

Charts must support:

* date range selection
* comparison between two periods
* zoom and hover values
* highlight abnormal spikes
* display average trend line

Add optional features:

* moving average line
* target line (KPI objective)
* alert markers when KPI exceeds threshold
